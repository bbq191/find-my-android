
将“配置页”与“地图选点页”合并，是符合现代移动端 GIS（地理信息系统）应用设计趋势的。iOS 的 Find My、Uber、滴滴出行，实际上都是在**同一个地图界面**上完成所有配置的。

### 为什么有必要合并？（Why）

1. **上下文不丢失 (Context is King):**
* **现状:** 图1 是纯文字配置，用户选“离开时”，却看不到“离开哪儿”的具体范围。图2 是纯地图，用户选了点，却忘了“我是要设到达还是离开？”
* **合并后:** 用户看着地图上的蓝色圆圈，一边拖动滑块调整半径，一边选择“到达/离开”。**“所见即所得”**。


2. **减少跳转 (Reduce Friction):**
* **现状:** 选位置 -> 跳页面 -> 选点 -> 确认 -> 回来 -> 保存。
* **合并后:** 拖动地图 -> 下方选条件 -> 保存。一步到位。


3. **利用 S24 Ultra 大屏优势:**
* S24 Ultra 的屏幕足够大，完全容得下“全屏地图 + 半屏控制面板”，分开反而浪费了屏幕空间。



---

### 如何合并？（The "All-in-One" Solution）

我们采用 **“全屏地图底座 + 悬浮控制面板”** 的架构。

#### 1. 界面架构 (Layout Architecture)

想象一下，不再有两个页面，而是一个页面包含两个层级：

* **层级 A (底层):** 全屏交互式地图 (Interactive Map)。
* 中心固定一个 **Pin (大头针)**。
* 绘制当前的 **电子围栏圆圈**。


* **层级 B (顶层):** 可交互的底部抽屉 (Persistent/Modal Bottom Sheet)。
* 承载所有的配置选项（通知类型、半径、频率）。



#### 2. 具体交互设计 (Detailed Interaction)

**默认状态 (进入页面时):**

* **地图:** 自动聚焦到目标（如马瑞雯）的当前位置。
* **Pin:** 悬浮在屏幕中心。
* **抽屉:** 处于 **半展开 (Half-expanded)** 状态，展示核心信息。

**底部抽屉的内容布局 (从上到下):**

1. **地址栏 (Address Header):**
* 显示当前 Pin 下方的地址（如“国茶港”）。
* 右侧放一个“搜索”图标，点击可搜索跳转。


2. **快捷位置切换 (Location Chips):**
* 一排横向滚动的胶囊按钮：
* `[定位到人]` `[定位到我]` `[家]` `[公司]`
* *点击这些胶囊，地图自动平移，Pin 指向该位置。*


3. **触发条件 (Trigger Segment):**
* 分段按钮：`[到达]` `[离开]`。


4. **半径滑块 (Radius Slider):**
* 直接在这里拖动，地图上的圆圈实时缩放。


5. **频次开关 (Frequency):**
* `Switch`: 仅通知一次。


6. **保存按钮:**
* 固定在底部。



#### 3. 动态交互演示

* **当用户拖动地图时:**
* 底部抽屉依然显示，但 Pin 保持在屏幕中心不动。
* 地图移动，Pin 下方的地址栏实时更新（“正在加载...” -> “昆明体育馆”）。


* **当用户调整半径时:**
* 地图上的蓝色半透明圆圈以 Pin 为圆心，实时变大变小。
* 用户能直观看到：“哦，500米半径正好覆盖了这个街区”。


### 总结

**合并非常有必要。**

请采用 **“地图即背景，配置即浮窗”** 的沉浸式设计。这不仅减少了 50% 的点击次数，还能让用户在设置围栏时拥有“上帝视角”的掌控感。
